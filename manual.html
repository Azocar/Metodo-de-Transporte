1. Fundamento Teórico

📌 Definición — Problema de Transporte

El Problema de Transporte es un modelo de programación lineal que busca determinar cómo distribuir un bien único desde m orígenes (almacenes, fábricas) hacia n destinos (sucursales, clientes) al mínimo costo total, respetando la capacidad de oferta de cada origen y el requerimiento de demanda de cada destino.

1.1 Elementos del Modelo

⚙️ Variables

xij = unidades a transportar del origen i al destino j

cij = costo unitario de transporte de i a j

📊 Parámetros

si = oferta disponible en el origen i

dj = demanda requerida en el destino j

1.2 Función Objetivo y Restricciones

Minimizar Z = Σ Σ cij · xij (para todo i, j) Sujeto a: Σ xij = si (restricción de oferta – fila) Σ xij = dj (restricción de demanda – columna) xij ≥ 0

1.3 Condición de Equilibrio

Para que el problema tenga solución directa, la suma de toda la oferta debe ser igual a la suma de toda la demanda:

Σ si = Σ dj → Problema Equilibrado
⚠️ Problema No Equilibrado Si la oferta total ≠ demanda total, se agrega una fila o columna ficticia (dummy) con costo 0 para equilibrar el modelo antes de aplicar cualquier método.

1.4 ¿Qué es el Método de la Esquina Noroeste?

Es una heurística constructiva que genera una Solución Básica Factible Inicial (SBFI) para el Problema de Transporte. No garantiza el óptimo, pero proporciona un punto de partida válido para métodos de optimización posteriores (MODI, Stepping-Stone).

✅ ¿Por qué se llama «Esquina Noroeste»? Porque siempre comienza asignando en la celda superior-izquierda de la tabla — la esquina que en cartografía corresponde al noroeste — y va avanzando hacia la derecha (→) o hacia abajo (↓) según cuál restricción se satisfaga primero.

2. Procedimiento Matemático Paso a Paso

2.1 Algoritmo Formal

  1. Construir la tabla de transporte. Ubicar los m orígenes en filas y los n destinos en columnas. Registrar los costos unitarios cij, la oferta si (columna derecha) y la demanda dj (fila inferior).
  2. Verificar el equilibrio. Calcular Σ si y Σ dj. Si son iguales → continuar. Si difieren → agregar fila o columna ficticia con costo 0.
  3. Posicionarse en la Esquina Noroeste. La celda actual comienza siempre en (i=1, j=1).
  4. Asignar el mínimo entre oferta y demanda actuales. xij = min( si , dj ) Registrar esta asignación en la celda actual.
  5. Actualizar oferta y demanda. Restar el valor asignado: si = si − xij y dj = dj − xij.
  6. Decidir el movimiento:
    • Si si = 0 y dj > 0 → mover hacia abajo (↓): i = i + 1
    • Si dj = 0 y si > 0 → mover hacia la derecha (→): j = j + 1
    • Si ambos son 0 → mover diagonal (↓→) solo si quedan celdas por visitar.
  7. Repetir pasos 4–6 hasta que se hayan satisfecho toda la oferta y toda la demanda (todas las si = 0 y todas las dj = 0).
  8. Calcular el Costo Total: Z = Σ Σ cij · xij (solo celdas asignadas)

2.2 Ejemplo Resuelto

Tres almacenes (S1, S2, S3) abastecen a cuatro destinos (D1, D2, D3, D4):

Origen \ Destino D1 D2 D3 D4 Oferta (si)
S1 2 3 1 5 120
S2 7 4 6 2 80
S3 4 3 8 1 80
Demanda (dj) 150 70 60 0 280 = 280 ✓
✅ Equilibrio verificado: Oferta total = 120 + 80 + 80 = 280 | Demanda total = 150 + 70 + 60 + 0 = 280

3.2 Registro de Asignaciones (Paso a Paso)

Paso Celda Min(si, dj) Asignación Costo Unit. Subtotal Movimiento
01 S1→D1 min(120, 150) = 120 120 u. $2 $240 ↓ (s1 = 0)
02 S2→D1 min(80, 30) = 30 30 u. $7 $210 → (d1 = 0)

04 S3→D2 min(80, 20) = 20 20 u. $3 $60 → (d2 = 0)

3.3 Cálculo del Costo Total

Z = (120×2) + (30×7) + (50×4) + (20×3) + (60×8) + (0×1) = 240 + 210 + 200 + 60 + 480 + 0 = $1,190
✅ Resultado: La Solución Básica Factible Inicial obtenida mediante el Método de la Esquina Noroeste tiene un Costo Total de Transporte = $1,190. Este valor puede mejorarse con el método MODI o Stepping-Stone, pero constituye un punto de partida válido para el problema.
📐 Número de asignaciones básicas: Un problema m×n equilibrado siempre producirá exactamente m + n − 1 asignaciones básicas. En este caso: 3 + 4 − 1 = 6 celdas asignadas ✓

4. Uso de la Aplicación Web

🌐 Tecnología utilizada: La aplicación fue desarrollada completamente en HTML5 + CSS3 + JavaScript Vanilla (sin frameworks externos). Se ejecuta directamente en el navegador abriendo el archivo index.html. No requiere instalación ni conexión a internet.

4.1 Guía de Uso — Paso a Paso

  1. Abrir la aplicación. Hacer doble clic en el archivo index.html para abrirlo en el navegador web (Google Chrome, Firefox o Edge recomendados).
  2. Configurar el tamaño del problema. En el panel "Configuración de la Tabla", introducir el número de Almacenes (Orígenes) y Sucursales (Destinos) deseados (entre 2 y 8). Presionar el botón 🗂️ Generar Tabla.
  3. Ingresar los datos del problema. La tabla aparecerá con datos de ejemplo precargados. Modificarlos según el problema a resolver: costos unitarios cij, oferta si (columna morada) y demanda dj (fila amarilla).
  4. Calcular la solución. En el panel de resultados, presionar ▶️ Calcular Ruta Óptima. La aplicación verificará automáticamente el equilibrio del problema y ejecutará el algoritmo de la Esquina Noroeste.
  5. Interpretar los resultados. Las celdas asignadas se resaltan con un badge mostrando las unidades asignadas. El panel de resultados muestra el Costo Total y el Registro de Asignaciones con cada paso del algoritmo, incluyendo el cálculo de costo por asignación.
  6. Limpiar y reiniciar. Presionar 🔄 Limpiar Resultados para borrar los resultados sin modificar los datos de entrada, o volver al paso 1 para un nuevo problema.

4.2 Lenguaje de Programación — JavaScript

JavaScript es un lenguaje de programación interpretado, orientado a objetos y de alto nivel, nativo de los navegadores web. En esta aplicación se utilizó JavaScript puro (Vanilla JS) para implementar toda la lógica del algoritmo sin dependencias externas.
